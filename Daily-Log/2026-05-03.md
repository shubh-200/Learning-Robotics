## 📝 What I Learned
### Key Concepts

- USB forwarding from windows to WSL kernel

### Gotchas / Things That Tripped Me Up
- USB forwarding was needed since WSL cannot capture USB directly
- systemd does not know that there is a monitor since it is a background daemon which runs headless. So, we need to tell it in the .service script where to open the display. 
- In real deployments, running the gui like this is not recommended. Only sensor drivers, planners, and algorithms are launched this way. For visualizing the robot's state, we run the viz locally and connect to the robot over the network via DDS.
